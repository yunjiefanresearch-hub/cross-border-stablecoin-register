# Stablecoin Rail Register
*(working title тАФ see "Naming" below)*

An **open, versioned, machine-readable** register of how jurisdictions regulate the
**permitted-activity / yield boundary on stablecoin payment rails**, with a
**corridor-level interoperability layer**. Built from primary sources, citable by DOI.

**Focus:** US ┬╖ Hong Kong ┬╖ EU ┬╖ UK ┬╖ Singapore ┬╖ Mainland China  ┬╖  **Doctrinal anchor:** United States (CLARITY Act ┬з404 / GENIUS Act)
**Backfill:** Brazil (corridor-only), Taiwan (parked)  ┬╖  **Status:** v0.2.0-dev (in active build) ┬╖ **Cadence:** quarterly diffs + event patches ┬╖ **License:** CC-BY-4.0 (data) / Apache-2.0 (code)

---

## What this is тАФ and what it is not
This is **not** another broad, shallow, proprietary "200-country" tracker, and it is **not** a scrape.
It is a focused, **depth-first** reference whose value is the **judgment in the data** тАФ sourcing and
ambiguity resolution made transparent (see [`METHODOLOGY.md`](METHODOLOGY.md)) тАФ and whose
differentiator is the **corridor layer**: what actually clears, and what breaks, at each regulatory
boundary along a cross-border flow (see [the corridor file](hk-br-usd-stablecoin-settlement.yaml)).

Each record is one verified `(jurisdiction ├Ч instrument ├Ч dimension)` fact: structured, sourced to a
primary instrument with a pinpoint, dated, version-stamped, and machine-readable against a published
[schema](record.schema.json).

## Naming
`Stablecoin Rail Register` is a working title. Criteria for the final name: sounds like
**infrastructure / a standard** (register, observatory), short, abbreviable, domain available, and
**not** colliding with other projects. The name lives in three places: this README, `CITATION.cff`,
and `ROADMAP.md`.
